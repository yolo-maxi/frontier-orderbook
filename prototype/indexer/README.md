# Frontier minimal indexer

A dependency-free indexer that reconstructs Frontier order-book state from
**logs alone** — the exact surface `eth_getLogs` exposes (address, topics, data,
blockNumber, logIndex), with **no contract storage reads**. It exists to answer a
single question empirically: *are the events Frontier emits sufficient to build an
indexer?* See [`../../docs/indexer-event-gaps.md`](../../docs/indexer-event-gaps.md)
for the findings.

## Layout

- `decode.mjs` — manual ABI log decoding (topic0 → event, then a 32-byte-word walk;
  all Frontier event fields are static, so no dependencies are needed).
- `indexer.mjs` — the engine: folds logs into markets / positions / trades / claims /
  cancels, then reconciles against a ground-truth file and prints a field-by-field
  report.
- `fixtures/` — generated by `prototype/test/IndexerFixture.t.sol`:
  - `scenario-{nofee,fee}.logs.json` — recorded logs (eth_getLogs shape).
  - `scenario-{nofee,fee}.truth.json` — what an indexer *should* recover, captured
    from the running contracts (sides, takers, exact amounts, payout tokens,
    per-position progress).

The `nofee` fixture is the important one: the **default deployment is zero-fee**,
which suppresses the `MakerFee`/`TakerFee` events, so it is the true test of the
event surface.

## Run

```bash
# from prototype/
forge test --match-contract IndexerFixtureGen   # (re)generate fixtures
node indexer/indexer.mjs scenario-nofee
node indexer/indexer.mjs scenario-fee
```

## Report legend

- **RECOVERED** — derived purely from events (no curve, no storage).
- **DERIVABLE\*** — not in any event; reconstructable only by replaying events
  *and* reimplementing the on-chain geometric curve + integer rounding. Flagged
  because it is a wei-exactness / maintenance hazard.
- **MISSING** — not recoverable from events at all.

With the additive `Deposit.isBid` flag and the `Swept` summary event (this branch),
both fixtures report **26 recovered / 0 missing**. Before those changes the
zero-fee deploy reported 10 recovered / 11 missing.

## How the gaps were closed (quick reference)

| Need | Source |
|---|---|
| order side (ask/bid) | `Deposit.isBid` |
| claim token, cancel token interpretation | position side (via `Deposit.isBid`), keyed by `positionId` |
| taker, amount in/out, post-sweep tick | `Swept(taker, tickBefore, tickAfter, amountIn, amountOut, takerFee)` |
| per-position unfilled principal | replay `Swept.tickAfter` (= frontier) vs deposit ordering, curve-free |
